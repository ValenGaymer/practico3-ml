Sobre los datos#

El conjunto de datos utilizado en este ejercicio contiene informaci√≥n obtenida de sensores inerciales y magn√©ticos port√°tiles durante la ejecuci√≥n de ejercicios de fisioterapia. Incluye ocho tipos de ejercicios, cada uno realizado en tres modalidades diferentes (correcta, r√°pida y de baja amplitud), repetidos varias veces por cinco sujetos. Los participantes usaron cinco unidades de sensores MTx fabricadas por XSens, cada una equipada con aceler√≥metro, giroscopio y magnet√≥metro tri-axiales, muestreados a 25 Hz. Los datos fueron recolectados bajo la aprobaci√≥n del Comit√© de √âtica de la Universidad de Bilkent, garantizando el anonimato de los participantes y con su consentimiento informado. Este conjunto de datos respalda investigaciones sobre la detecci√≥n automatizada y evaluaci√≥n de ejercicios de fisioterapia.

Estructura de los datos#

La estructura de los datos est√°n distribuidos en carpetas de la siguiente manera:

  • s1, ‚Ä¶, s5: representan los sujetos 1 a 5.

  • e1, ‚Ä¶, e8: representan los 8 tipos de ejercicios.

  • u1, ‚Ä¶, u5: representan las 5 unidades de sensores utilizadas.

La organizaci√≥n de los archivos es la siguiente:

  1. Sesiones de plantilla (template_session.txt): Para cada sujeto, tipo de ejercicio y unidad de sensor, este archivo contiene datos recolectados durante una sesi√≥n de grabaci√≥n que incluye tres ejecuciones para cada uno de los tres tipos de ejecuci√≥n (correcta, r√°pida y de baja amplitud).   2. Intervalos de tiempo de plantillas seleccionadas (template_times.txt): Estos archivos, disponibles para cada sujeto y tipo de ejercicio, especifican los √≠ndices de tiempo de inicio y fin de las plantillas seleccionadas en los archivos template_session.txt.

  3. Sesiones de prueba (test.txt): Para cada sujeto, tipo de ejercicio y unidad de sensor, este archivo contiene datos de una sesi√≥n de prueba con 10 repeticiones para cada tipo de ejecuci√≥n.

Cada fila de los archivos template_session.txt y test.txt incluye el √≠ndice de tiempo (ùëõ = 1, 2, ‚Ä¶ , ùëÅ) y las mediciones en los ejes ùë•, ùë¶ y ùëß de los sensores de aceler√≥metro (acc), giroscopio (gyr) y magnet√≥metro (mag), cuyo orden se indica en las filas de encabezado. Descomposici√≥n estacional#

La descomposici√≥n estacional es un m√©todo que permite modelar los factores estacionales de una serie temporal. Se utiliza para descomponer una serie temporal en sus componentes de tendencia, estacional y resto. La estacionalidad es la repetici√≥n de variaciones en una variable cada cierto per√≠odo, que suele ser igual o menor a un a√±o. Los fen√≥menos estacionales pueden ocurrir diariamente en una hora determinada, todos los d√≠as, o en un mes determinado a cada a√±o.

Debido a lo extenso que resultar√≠a el an√°lisis si se considerase cada uno de las combinaciones entre sujetos, ejercicios y unidad, se decidi√≥ fijar la descomposici√≥n a un sujeto y un ejercicios: el primero, el n√∫mero de unidades se mantuvo.

def temp_decompose(sujeto, ejercicio, unidad, variable_set, freq=30):
    """
    Genera la descomposici√≥n temporal para las variables asociadas (x, y, z) de un sujeto, ejercicio y unidad.

    Par√°metros:
    sujeto (int): N√∫mero del sujeto.
    ejercicio (int): N√∫mero del ejercicio.
    unidad (int): N√∫mero de la unidad.
    variable_set (str): Puede ser 'gyr', 'acc' o 'mag'.
    freq (int): Frecuencia esperada de estacionalidad (por defecto 30).

    Retorna:
    None. Muestra las gr√°ficas de descomposici√≥n temporal.
    """
    key = f's{sujeto}e{ejercicio}u{unidad}'
    if key not in templates:
        print(f"Datos para la clave {key} no encontrados.")
        return

    data = templates[key]
    variables = [f"{variable_set}_x", f"{variable_set}_y", f"{variable_set}_z"]

    fig, axes = plt.subplots(4, 3, figsize=(18, 12), sharex=True)
    fig.suptitle(f"Descomposici√≥n Temporal - Sujeto {sujeto}, Ejercicio {ejercicio}, Unidad {unidad}, Variables: {variable_set}", fontsize=18)

    for i, var in enumerate(variables):
        if var not in data.columns:
            print(f"Variable {var} no encontrada en los datos.")
            continue
        decomposition = seasonal_decompose(data[var], model='additive', period=freq, extrapolate_trend='freq')

        axes[0, i].plot(decomposition.observed, label='Observado', color='purple')
        axes[0, i].set_title(f"{var} - Observado", fontsize=12)
        axes[0, i].legend()

        axes[1, i].plot(decomposition.trend, label='Tendencia', color='orange')
        axes[1, i].set_title(f"{var} - Tendencia", fontsize=12)
        axes[1, i].legend()

        axes[2, i].plot(decomposition.seasonal, label='Estacionalidad', color='green')
        axes[2, i].set_title(f"{var} - Estacionalidad", fontsize=12)
        axes[2, i].legend()

        axes[3, i].plot(decomposition.resid, label='Residuo', color='red')
        axes[3, i].set_title(f"{var} - Residuo", fontsize=12)
        axes[3, i].legend()

    plt.tight_layout(rect=[0, 0, 1, 0.94])
    plt.show()

Test de Dickey - Fuller#

El Test de Dickey-Fuller (o prueba de Dickey-Fuller) es una prueba estad√≠stica ampliamente utilizada en series temporales para verificar si una serie es estacionaria. En t√©rminos simples, se utiliza para evaluar si una serie tiene una ra√≠z unitaria, lo cual implica que la serie no es estacionaria.

¬øQu√© significa ‚Äúestacionaria‚Äù?#

Una serie temporal es estacionaria si sus propiedades estad√≠sticas, como la media, la varianza y la autocovarianza, no cambian con el tiempo. Esto es importante porque muchos modelos de series temporales, como el ARIMA, requieren que los datos sean estacionarios.

Hip√≥tesis del test de Dickey-Fuller#

  • Hip√≥tesis nula ((H_0)): La serie tiene una ra√≠z unitaria (no es estacionaria).

  • Hip√≥tesis alternativa ((H_a)): La serie es estacionaria. El objetivo del test es rechazar (H_0) para concluir que la serie es estacionaria.

Resultados#

El test produce:

  1. Estad√≠stico de prueba: Un valor que se compara con los valores cr√≠ticos.

  2. Valores cr√≠ticos: Dependen del nivel de confianza (1%, 5%, 10%).

  3. p-value: Probabilidad asociada a la hip√≥tesis nula. Si (p < 0.05), se rechaza (H_0).

Interpretaci√≥n#

  • Si se rechaza (H_0): La serie es estacionaria. Puede usarse directamente en modelos como ARIMA.

  • Si no se rechaza (H_0): La serie no es estacionaria. Se pueden aplicar transformaciones (como diferencias) para lograr estacionariedad.

ACF y PACF#

En el an√°lisis de series temporales, las gr√°ficas de la Funci√≥n de Autocorrelaci√≥n (ACF) y de la Funci√≥n de Autocorrelaci√≥n Parcial (PACF) son herramientas clave para identificar la estructura subyacente de una serie temporal. Mientras que la ACF muestra la correlaci√≥n de una serie con sus propios valores en diferentes retardos (lags), la PACF muestra esta correlaci√≥n eliminando los efectos de los retardos intermedios.

Funci√≥n de Autocorrelaci√≥n (ACF)#

La ACF es una representaci√≥n gr√°fica que muestra la correlaci√≥n de una serie temporal con ella misma en diferentes retardos. El coeficiente de correlaci√≥n mide qu√© tan relacionadas est√°n dos variables.

  • Un coeficiente de correlaci√≥n de 1 indica una relaci√≥n positiva perfecta.

  • Un coeficiente de -1 indica una relaci√≥n negativa perfecta.

  • Un coeficiente de 0 indica que no hay relaci√≥n entre las variables.

La gr√°fica de la ACF es √∫til para identificar el orden de un modelo AR (Autorregresivo). El orden del modelo AR corresponde al n√∫mero de retardos que deben incluirse en el modelo. Los picos en la gr√°fica de la ACF indican los retardos que tienen correlaciones significativas y, por lo tanto, deben ser considerados en el modelo.

Funci√≥n de Autocorrelaci√≥n Parcial (PACF)#

La PACF es una representaci√≥n gr√°fica que muestra la correlaci√≥n de una serie temporal con sus valores en diferentes retardos, eliminando los efectos de los retardos previos. A diferencia de la ACF, la PACF a√≠sla el efecto directo de cada retardo. La PACF se utiliza para identificar el orden de un modelo MA (Media M√≥vil). El orden del modelo MA corresponde al n√∫mero de retardos que deben incluirse en el modelo. Los picos significativos en la gr√°fica de la PACF indican los retardos que son relevantes para el modelo.

def seleccionar_mejores_modelos(templates=templates):
    modelos = ['knn', 'lineal', 'lasso', 'ridge', 'random_forest', 'xgboost', 'svr']
    variables = ['gyr_x', 'gyr_y', 'gyr_z', 'acc_x', 'acc_y', 'acc_z', 'mag_x', 'mag_y', 'mag_z']
    mejores_resultados = {modelo: {'Variable': None, 'R2': -float('inf'), 'Unidad': None, 'Ejercicio': None, 'Hiperpar√°metros': None, 'Ljung-Box p-value': None, 'Jarque-Bera p-value': None} for modelo in modelos}
    for modelo in modelos:
        resultados_modelo = globals()[f"{modelo}_varmodel"]
        for i, variable in enumerate(variables):
            for resultado in resultados_modelo[i]:
                r2_val = resultado['R^2']
                if r2_val > mejores_resultados[modelo]['R2']:
                    mejores_resultados[modelo] = {
                        'Variable': variable,
                        'R2': r2_val,
                        'Unidad': resultado['Unidad'],
                        'Ejercicio': resultado['Ejercicio'],
                        'Separaci√≥n':resultado['Separaci√≥n'],
                        'Ventana':resultado['Segundos'],
                        'Hiperpar√°metros': resultado['Hiperpar√°metros'],
                        'Ljung-Box p-value': resultado.get('Ljung-Box p-value', None),
                        'Jarque-Bera p-value': resultado.get('Jarque-Bera p-value', None)
                    }
    df_mejores_modelos = pd.DataFrame.from_dict(mejores_resultados, orient='index').drop(columns = ['R2', 'Ljung-Box p-value', 'Jarque-Bera p-value'])
    df_mejores_modelos.reset_index(inplace=True)
    df_mejores_modelos.rename(columns={'index': 'Modelo'}, inplace=True)
    return df_mejores_modelos

def analizar_residuos(df_test: pd.DataFrame, df_train: pd.DataFrame):
    def calcular_residuos(df: pd.DataFrame):
        if df.isnull().values.any():
            print(df.isnull().sum())
            df = df.dropna()

        df['Residuos'] = df[1] - df[2]

        mae = mean_absolute_error(df[1], df[2])
        mse = mean_squared_error(df[1], df[2])
        rmse = np.sqrt(mse)
        mape = np.mean(np.abs(df['Residuos'] / df[1])) * 100
        r2 = r2_score(df[1], df[2])

        ljung_box = acorr_ljungbox(df['Residuos'], lags=[10], return_df=True)
        ljung_box_pvalue = ljung_box['lb_pvalue'].iloc[0]

        jarque_bera_result = jarque_bera(df['Residuos'])
        jarque_bera_pvalue = jarque_bera_result[1]

        tabla_resumen = pd.DataFrame({
            'MAPE (%)': [mape],
            'MAE': [mae],
            'MSE': [mse],
            'RMSE': [rmse],
            'R2': [r2],
            'Ljung-Box p-value': [ljung_box_pvalue],
            'Jarque-Bera p-value': [jarque_bera_pvalue]
        })

        return df, tabla_resumen

    def generar_graficos_lado_a_lado(df_train: pd.DataFrame, df_test: pd.DataFrame):
        fig, axes = plt.subplots(2, 2, figsize=(18, 8))

        qqplot(df_train['Residuos'], line='s', ax=axes[0, 0])
        axes[0, 0].set_title('QQPlot (Entrenamiento)')

        qqplot(df_test['Residuos'], line='s', ax=axes[0, 1])
        axes[0, 1].set_title('QQPlot (Test)')

        plot_acf(df_train['Residuos'], ax=axes[1, 0], lags=30)
        axes[1, 0].set_title('ACF (Entrenamiento)')

        plot_acf(df_test['Residuos'], ax=axes[1, 1], lags=30)
        axes[1, 1].set_title('ACF (Test)')

        plt.tight_layout()
        plt.show()

    df_train, tabla_train = calcular_residuos(df_train)

    # Calcular residuos y m√©tricas para el conjunto de test
    df_test, tabla_test = calcular_residuos(df_test)

    # Generar gr√°ficos lado a lado (solo QQPlot y ACF)
    generar_graficos_lado_a_lado(df_train, df_test)

    return {"Entrenamiento": tabla_train, "Test": tabla_test}
